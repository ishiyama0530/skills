# Style Profile Examples

## technical_article

```markdown
- Formality: 常体または軽い敬体。混在は避ける。
- Point of view: 一人称は少なめ。ただし失敗例では使える。
- Opening style: 結論、制約、失敗例から入る。
- Paragraph density: 短め。1段落1トピック。
- Vocabulary: 技術用語は許容。抽象的なビジネス語は抑える。
- Assertion strength: 条件を明示して断言する。
- Avoid: 偽の共感、過剰な見出し、テンプレまとめ。
```

## blog_note

```markdown
- Formality: 常体または柔らかい敬体。
- Point of view: 一人称を使える。
- Opening style: 違和感、体験、記憶、観察から入る。
- Paragraph density: 短めでもよい。余白を残せる。
- Vocabulary: 雰囲気語に頼りすぎない。
- Emotional temperature: 低〜中。場面に紐づく感情だけ入れる。
- Avoid: 学びの押し売り、テンプレ結論、抽象語の連発。
```

## business_document

```markdown
- Formality: 敬体。
- Point of view: 必要に応じて「弊社」「当社」「私」を使う。
- Opening style: 結論、依頼、目的から入る。
- Paragraph density: 箇条書きを許容。読み手の判断負荷を下げる。
- Vocabulary: 正確性優先。曖昧な美辞麗句を避ける。
- Assertion strength: 判断と根拠を分ける。
- Avoid: 余談、過剰なクッション、根拠のない断定。
```
