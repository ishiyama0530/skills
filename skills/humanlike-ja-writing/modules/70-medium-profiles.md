# Medium Profiles

## 目的

媒体ごとに自然な文体を定義します。AI臭の少ない文章は、媒体に合っている必要があります。

## technical_article

対象:

- Zenn
- Qiita
- 技術ブログ
- README
- 設計メモ

自然な特徴:

- 結論、前提、制約を先に出す。
- 失敗例、比較、コード、設定値を使う。
- 主張は強くてよいが、条件を明示する。
- 「何をしたらどうなったか」を書く。

避ける:

- 大げさな見出し。
- 偽の共感。
- 「近年、注目されています」のような背景作り。
- 感情過多。
- 余韻だけの結論。

冒頭例:

```text
Next.jsのキャッシュは、まず「どこでキャッシュされているか」を分けないと混乱する。
```

## business_document

対象:

- 社内共有
- 提案文
- 報告書
- 議事録
- 稟議の下書き

自然な特徴:

- 目的、判断材料、結論、依頼、期限を明確にする。
- 感情より、読み手の判断負荷を下げる。
- 具体的な次アクションを書く。
- 主語と責任範囲を曖昧にしない。

避ける:

- くだけすぎた表現。
- 余談。
- 根拠のない断定。
- 長い前置き。

冒頭例:

```text
結論として、今月中にA案で進めたいです。理由は、既存実装への影響が一番小さいためです。
```

## blog_note

対象:

- note
- 個人ブログ
- エッセイ寄りの記事

自然な特徴:

- 一人称、違和感、体験、発見を使える。
- 完全に整理しきらない余白を残してよい。
- ただし、抽象語だけで雰囲気を作らない。
- 感情は場面に紐づける。

避ける:

- テンプレまとめ。
- 過剰な見出し。
- 「学び」「気づき」の水増し。
- 何でも「本質」に寄せること。

冒頭例:

```text
AIが書いた文章を読んでいて一番引っかかるのは、語尾よりも「誰もそこにいない感じ」だった。
```

## social_post

対象:

- X
- LinkedIn
- 短文投稿

自然な特徴:

- 1つの主張に絞る。
- 抽象論より、具体的な一場面を出す。
- 短くても断言範囲を守る。
- 過剰な箇条書きにしない。

避ける:

- 「〇〇な人へ」テンプレ。
- バズ狙いの煽り。
- 偽の教訓。
- 体験談の捏造。

冒頭例:

```text
AIに「自然に書いて」と頼むと、だいたい自然じゃない。先に渡すべきなのは文体ではなく、判断材料。
```

## marketing_copy

対象:

- LP
- メルマガ
- サービス紹介
- キャンペーン文

自然な特徴:

- 読者の課題、具体的な利用場面、ベネフィットを書く。
- ただし、読者の悩みを勝手に代弁しすぎない。
- 数字や事例がある場合は優先する。
- CTAは自然に置く。

避ける:

- 「こんなお悩みありませんか」の連発。
- 「画期的」「革新的」「次世代」の乱用。
- 実績の捏造。
- すべてをポジティブに言い切ること。

## email_message

対象:

- 業務メール
- 依頼文
- お礼
- 断り
- 日程調整

自然な特徴:

- 用件を先に書く。
- 相手に必要な情報だけを出す。
- 依頼、期限、確認事項を明確にする。
- 丁寧だが、回りくどくしない。

避ける:

- 過剰なクッション言葉。
- 長い謝辞。
- 曖昧な依頼。
- 「ご確認いただけますと幸いです」の乱用。

## medium selection

媒体が不明な場合は、内容から仮定します。

| 内容 | 仮定する媒体 |
|---|---|
| コード、技術選定、エラー | technical_article |
| 社内報告、依頼、判断 | business_document |
| 体験、考察、違和感 | blog_note |
| 短い主張、告知 | social_post |
| 商品、サービス訴求 | marketing_copy |
| 宛先、返信、依頼 | email_message |

不確実な場合は、仮定を明示します。
