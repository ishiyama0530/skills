# Review Rubric

## 目的

既存文または初稿のAI臭を、安定してレビューするための採点基準です。採点は `pass / warn / fail` で行います。

## Decision

```text
pass: そのまま出してよい。軽微な表層修正のみ。
rewrite-recommended: warnが複数ある。リライト推奨。
rewrite-required: failがある。リライト必須。
need-more-context: P0/P1の材料不足で、捏造なしには直せない。
```

## 判定軸

### 1. stance

| 判定 | 基準 |
|---|---|
| pass | 主張、立場、断言範囲が明確 |
| warn | 主張はあるが一般論が多い |
| fail | 誰の立場か、何を言いたいか分からない |

### 2. specificity

| 判定 | 基準 |
|---|---|
| pass | 数字、固有名詞、場面、体験、比較のいずれかがある |
| warn | 具体例はあるが弱い、または一般化されすぎている |
| fail | 抽象論だけで、どの文脈にも当てはまる |

### 3. agency

| 判定 | 基準 |
|---|---|
| pass | 誰が何を見て、何を判断したか分かる |
| warn | 主体はあるが一部曖昧 |
| fail | 無生物主語、受け身、抽象名詞が多く、行為者が見えない |

### 4. causality

| 判定 | 基準 |
|---|---|
| pass | 因果の中間メカニズム、前提、条件がある |
| warn | 因果はあるが説明が薄い |
| fail | 「これにより」「その結果」だけでつながっている |

### 5. structure

| 判定 | 基準 |
|---|---|
| pass | 媒体に合った構成で、テンプレ感が少ない |
| warn | 一部に定型導入や定型結論がある |
| fail | 全体が導入、背景、課題、解決策、まとめのテンプレに見える |

### 6. rhythm

| 判定 | 基準 |
|---|---|
| pass | 文長、段落、語尾に自然な揺れがある |
| warn | 少し均質だが読みにくくはない |
| fail | 全文が同じ調子で、機械的に整いすぎている |

### 7. vocabulary

| 判定 | 基準 |
|---|---|
| pass | 抽象語が具体的な動作や観察に支えられている |
| warn | 高リスク語が複数あるが、意味は通る |
| fail | 抽象語、AI頻出語、カタカナ語で水増ししている |

### 8. surface

| 判定 | 基準 |
|---|---|
| pass | 記号、Markdown癖、定型句が少ない |
| warn | 表層的なAI臭が少し残る |
| fail | 全角ダッシュ、太字コロン、テンプレ句が多く、明らかにAI出力っぽい |

## リライト条件

```text
- fail が1つでもある: rewrite-required
- P0/P1のfailがあり、材料が不足している: need-more-context
- warn が3つ以上ある: rewrite-recommended
- surface のwarnだけ: final passまたはlintで修正
- すべてpass、またはwarnが1〜2個: pass with minor edits
```

## レビュー出力の最小形式

```markdown
Decision: rewrite-required

Critical:
- P0 specificity: 具体材料がなく、一般論に見える。

Major:
- P1 causality: 「これにより」で因果をつないでいるが、中間説明がない。

Rewrite plan:
1. 主張を1文で固定する。
2. 数字または実際の場面を入れる。
3. 「これにより」を削り、因果の中間を書く。
```

## レビュー時の注意

- 表現の好みだけで過剰に直さない。
- ユーザーの文体を壊さない。
- 必要な専門用語まで削らない。
- 情報不足の場合、AIが勝手に具体例を作らない。
- 文章の目的が形式文書なら、人間味より明確性を優先する。
