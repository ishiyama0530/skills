# Drafting Workflow

## 目的

AI臭を出しにくい初稿を作るための手順です。初稿の時点で、主張、具体性、主体、因果、媒体適合を入れます。

## 標準手順

### 1. 読者と媒体を確認する

同じ内容でも、媒体によって自然な構成は違います。

- 技術記事: 結論、前提、制約、手順、比較。
- 業務文: 目的、判断材料、依頼、期限、次アクション。
- ブログ/note: 違和感、体験、発見、余韻。
- SNS: 具体的な一文、短い主張、読者の持ち帰り。

### 2. 主張を1文で固定する

本文を書く前に、内部で次を作ります。

```text
この文章で言いたいことは「...」である。
```

この一文が曖昧な場合、初稿は一般論になります。

### 3. 根拠を分類する

根拠を次に分けます。

```text
- 体験: 自分やチームが実際に経験したこと
- 観察: 実際に見た読者・ユーザー・現場の動き
- 数字: 測定値、件数、割合、時間
- 比較: Before/After、他案との違い
- 失敗: うまくいかなかったこと
- 出典: 確認可能な参照
```

根拠がない場合は、断定を弱め、一般論として書きます。

### 4. 冒頭を選ぶ

テンプレ導入を避けます。

避ける:

```text
本記事では、AIっぽくない文章を書くためのポイントを解説します。
```

代替:

```text
AIに「自然に書いて」と頼むだけでは、だいたい自然にならない。
```

冒頭候補:

- 結論から入る。
- 実際の失敗から入る。
- 数字から入る。
- 違和感から入る。
- 制約から入る。

### 5. 1段落1トピックで書く

1つの段落では1つの話だけを扱います。

- 最初の文で段落の役割が分かる。
- 次の文で根拠や例を出す。
- 最後の文で判断や接続を置く。

ただし、すべての段落を同じ長さにしないでください。

### 6. 主体を明示する

AI臭い文は、抽象名詞や無生物を主語にしがちです。

避ける:

```text
この結果は、課題を浮き彫りにしています。
```

代替:

```text
5人中4人が同じ画面で止まったので、初回利用者には見つけにくい配置だと判断した。
```

### 7. 因果を飛ばさない

「これにより」「その結果」でつないだだけでは、因果は説明できません。

悪い:

```text
AIを導入することで、生産性が向上します。
```

よい:

```text
議事録の整形に限れば、AIを使う効果は出やすい。人が30分かけていた整形作業を、レビュー込みで10分ほどに縮められるからだ。
```

### 8. リズムを調整する

- 長い説明のあとに短い判断を置く。
- 箇条書きと地の文を使い分ける。
- 同じ接続詞を連続させない。
- 語尾を機械的にそろえない。
- すべてを丁寧にまとめ切らない。

## 初稿で避けること

- 「本記事では」「この記事では」で始める。
- 「まず、次に、最後に」を無条件で使う。
- 何でも3つに分ける。
- 「近年、急速に、注目されています」で背景を作る。
- 「多くの人が」「専門家によると」を根拠なしに使う。
- 「重要です」「効果的です」で説明を終える。
- 「参考になれば幸いです」で締める。
- 根拠のない感情や余談を足す。

## Draft checklist

```text
[ ] 主張が1文で言える
[ ] 読者と媒体に合っている
[ ] 具体材料が最低1つある
[ ] 誰が何をしたか分かる
[ ] 因果に中間説明がある
[ ] 冒頭がテンプレではない
[ ] 結論が儀礼的な締めではない
[ ] 高リスク表現が残りすぎていない
```
