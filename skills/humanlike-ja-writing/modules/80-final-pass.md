# Final Pass

## 目的

最終出力前に、AI臭、事実誤認、媒体ズレを確認します。ここでは表層だけでなく、冒頭と結論も重点的に見ます。

## チェック順

### 1. 冒頭だけ読む

確認:

- 「本記事では」「この記事では」で始まっていないか。
- 主張、場面、数字、違和感のいずれかがあるか。
- 読者に不要な前置きがないか。

### 2. 結論だけ読む

確認:

- 「いかがでしたか」「参考になれば幸いです」で逃げていないか。
- 書き手の判断で終わっているか。
- 本文の主張と結論がずれていないか。

### 3. 抽象語だけ拾う

確認語:

```text
本質
解像度
示唆
浮き彫り
重要
価値
多面的
包括的
役割
可能性
```

対応:

- 具体的な動作、数字、場面、判断に置き換えられるか確認する。
- 残す場合は、周辺に具体材料があるか確認する。

### 4. 主語だけ拾う

確認:

- 誰が何をしたか分かるか。
- 「データが示す」「課題が浮き彫りになる」などの無生物主語が多すぎないか。
- 受け身が続いていないか。

### 5. 接続詞だけ拾う

確認語:

```text
また
さらに
加えて
一方で
これにより
その結果
つまり
なお
```

対応:

- 連続していれば削る。
- 「これにより」は、因果の中間を書く。
- 接続詞なしでも読める文は削る。

### 6. 記号だけ拾う

確認:

- 全角ダッシュ、エムダッシュ。
- 太字コロン箇条書き。
- 過剰な鉤括弧。
- Markdown残骸。
- `contentReference` や `oai_citation` のようなツール由来の残骸。

### 7. リズムを見る

確認:

- 全段落が同じ長さではないか。
- 全文が同じ語尾ではないか。
- 箇条書きが続きすぎていないか。
- 一文が長すぎないか。
- 短文ばかりでぶつ切りになっていないか。

### 8. 事実を確認する

確認:

- 数字、固有名詞、日付、出典が正しいか。
- 不確かなことを断言していないか。
- 最新情報が必要な内容を古い記憶で書いていないか。
- 体験談や観察を捏造していないか。

## Codex lint

ファイルとして確認できる場合は、次を実行します。

```bash
node skills/humanlike-ja-writing/tools/ja-ai-smell-lint.mjs <target-file>
```

厳しめに失敗判定したい場合:

```bash
node skills/humanlike-ja-writing/tools/ja-ai-smell-lint.mjs <target-file> --strict
```

## 最終判定

```text
pass:
  そのまま出せる。

minor-edit:
  表層修正だけで出せる。

rewrite:
  P0〜P2の問題が残っている。

need-more-context:
  具体材料が不足しており、捏造なしには自然にできない。
```

## 最終チェックリスト

```text
[ ] 冒頭がテンプレではない
[ ] 結論が儀礼的な締めではない
[ ] 主張が見える
[ ] 具体材料がある、または一般論として扱っている
[ ] 主体が明確
[ ] 因果が飛んでいない
[ ] 抽象語が具体材料に支えられている
[ ] 媒体に合った文体
[ ] 高リスク表現が残りすぎていない
[ ] 数字、出典、体験談を捏造していない
```
