# Rewrite Playbook

## 目的

AI臭を検出したあと、表層置換ではなく、原因に応じて文章を直すための手順です。

## 基本方針

1. P0/P1から直す。
2. 意味を変えない。
3. 事実、数字、経験を捏造しない。
4. 抽象語は具体的な行動、観察、数字、判断に戻す。
5. 媒体に合う範囲でリズムを調整する。
6. 最後に高リスク表現と記号を掃除する。

## 変換パターン

### 1. 書き手不在を直す

悪い:

```text
AIの活用は、今後ますます重要になるでしょう。
```

直し方:

- 誰の立場かを出す。
- どの範囲で重要なのかを限定する。
- 何を見てそう判断したのかを入れる。

改善:

```text
下書きや要約に限れば、AIを使わない理由はかなり減った。少なくとも、毎週の議事録整形に30分かける作業はもう戻したくない。
```

### 2. 具体材料なしを直す

悪い:

```text
この方法により、作業効率を高めることが可能です。
```

直し方:

- 作業名を入れる。
- Before/Afterを入れる。
- 数字がなければ、具体場面だけでも入れる。

改善:

```text
議事録の整形なら、この方法は効きやすい。話者名の統一、箇条書き化、次アクションの抽出までを一度に済ませられる。
```

### 3. 主体不明を直す

悪い:

```text
このデータは、ユーザー体験の課題を浮き彫りにしています。
```

改善:

```text
5人中4人が検索フォームの場所で止まった。初回利用者には、今の配置は見つけにくい。
```

### 4. 因果が薄い文を直す

悪い:

```text
AIを導入することで、チームの生産性が向上します。
```

改善:

```text
議事録の要約とタスク抽出に限定すれば、生産性は上がりやすい。人が毎回やっていた整形と抜け漏れ確認を、レビュー込みで短時間に寄せられるからだ。
```

### 5. テンプレ冒頭を直す

悪い:

```text
本記事では、AIっぽくない文章を書くためのポイントを解説します。
```

改善候補:

```text
AIに「自然に書いて」と頼むだけでは、だいたい自然にならない。
```

```text
AI臭い文章の正体は、語尾よりも書き手の不在にある。
```

```text
まず消すべきなのは「本記事では」ではない。文章の中に誰の判断もない状態だ。
```

### 6. 定型結論を直す

悪い:

```text
いかがでしたか。この記事が参考になれば幸いです。
```

改善:

```text
表現を少し崩すだけでは足りない。先に、誰が何を見てそう判断したのかを入れる必要がある。
```

### 7. 抽象語を直す

| 検出 | 直し方 |
|---|---|
| 本質 | 何のどの性質かを書く |
| 解像度 | 何がどれだけ具体化されたかを書く |
| 示唆 | 何をどこまで言えるかを書く |
| 浮き彫り | 誰が何を見て気づいたかを書く |
| 重要な役割 | 具体的に何をしているかを書く |
| 多面的 | 観点を列挙する |

### 8. 回りくどい表現を直す

| 検出 | 置換候補 |
|---|---|
| 〜することが可能です | できます / 具体的な動詞 |
| 〜につながります | 何がどう変わるかを書く |
| 〜として位置づけられます | 何であるかを直接書く |
| 〜の役割を果たします | 何をするかを書く |
| これにより | 削る。必要なら因果を直接書く |

## 感情と余談の扱い

感情は、実際の場面や判断に紐づく場合だけ入れます。

悪い:

```text
この機能を使って、とても嬉しくなりました。
```

改善:

```text
手作業で直していた改行が一発で揃った。地味だけど、ここで少し楽になった。
```

## リライト後チェック

```text
[ ] 主張が残っている
[ ] 意味を変えていない
[ ] 事実を増やしていない
[ ] 主体が明確になった
[ ] 因果が説明された
[ ] 抽象語が具体化された
[ ] テンプレ導入・結論が消えた
[ ] 媒体に合う文体になった
```
