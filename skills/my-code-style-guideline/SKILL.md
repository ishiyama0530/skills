# 共通コードスタイル

## 共通ルール
- **指示されたことにのみ注力し、余計な実装は含めない**
- **KISSの原則に従うこと**
    - 設計はシンプルであるほど良い
    - 不要な複雑さを避ける
    - 理解しやすく、保守しやすいものを作る
    - 過度なモジュール化や抽象化は避ける
- セキュリティベストプラクティスに従う
- 疎結合で高凝集な設計を心がける
- 冪等性を考慮した実装を心がける
- 既存のコードとの一貫性が保たれているか
- オプショナルな型は基本的に使用しない

## カプセル化
- 驚き最少の原則に従う
- 引数やグローバル変数に変更を加えない

## メソッド設計方針

メソッドの呼び出し構造はフラットに保つ。

```
❌ A → B → C → D（連鎖的な呼び出し）
✅ A → B, A → C, A → D（1つの起点から並列に呼び出す）
```

1つのメソッドが処理の起点となり、各処理を直接呼び出す構造を優先する。

例外のスローとロギングは起点となるメソッド（A）で一元的に処理する。B, C, Dでは例外を投げず、Aに結果を返す。
